ASP.NET Overview

  • ASP.NET is a web application framework developed and marketed by Microsoft to allow programmers to build dynamic web sites
  • ASP.NET allows you to use a full featured programming language such as C# or VB.Net to build web application easily
  • ASP.NET provides a programming model which is a comprehensive software infrastructure and various services required to build up robust web applications for the PC as well as mobile devices
  • ASP.NET works on top of the HTTP protocol and uses the HTTP commands and polices to set a browser to server bilateral communication and cop-operation
  • ASP.NET is part of Microsoft .Net platform
  • ASP.NET applications are compiled codes written using the extensible and reusable components or objects present in .Net framework
  • ASP.NET application codes can be written in any of the following languages
    • C#
    • Visual Basic .Net
    • Jscript
    • J#
  • ASP.NET is used to produce interactive, data-driven web applications over the Internet
  • ASP.NET consist of a large number of controls such as text boxes, buttons and labels fro assembling, configuring, and manipulating code to create HTML pages
  • Although a web site is simple, the process that eventually results in the page Default.aspx being displayed in your browser isn't so simple. ASP.BET page can't do much. It needs to be processed and served by a web server before the browser can display it. Thats why VS automatically started up by IIS Express to handle the request for the page
  • It is important to realize that the ASPX file you modified in VS is not the same as the one that eventually gets displayed by the browser
  • When you create a page in VS you add markup to it. The markup is an aspx page that is a combination of HTML code for ASP.NET Server controls, code written in Visual Basic.NET or C# and more
  • When you request as ASPX page in your browser the web server process the page, executes any server-side code it finds in the file, and effectiveness transforms the ASP.NET markup into plain HTML that it then sends to the browser, where it is displayed
  • When running the original ASPX an page you will see when looking under the source of the developer mode that it differs from the original code

Introduction

  • When you type a URL like www.wrox.com in your web browser and press Enter, the browser sends a request to the web server at that address
  • HTTP (HyperText Transfer Protocol) is a protocol by which web browsers and web servers communicate
  • When you request the URL you send a request to the server
  • When the server is active and the request is valid, the server accepts the request, processes it, and then sends the response back to the client browser
  • Because you are using the IIS Express, the server and the client are really the same machine
  • In a real world scenario, the website is hosted on an external web server where it can be accessed by many different clients
  • For static files like HTML files or images, the web server simply reads the file from its local hard drive and sends in to the browser
  • If the web server hands over the request to another piece of software that ables to process the page - the concept is called Application Mapping or Handler Mapping, where the extention to a file is mapped to an application that is capable of handling it
